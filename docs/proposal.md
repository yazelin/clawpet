# OpenClaw 寵物企劃提案書（MVP v1）

## 1) 企劃定位

### 專案暫名
- **clawpet**（可再命名）

### 核心願景
- 把「私人女友」的陪伴感，轉成更普適、溫暖、低負擔的「**AI 寵物陪伴**」。
- 在 OpenClaw 內讓寵物「活著」：可互動、可成長、可收藏、可分享。
- 先以 **貓咪** 為 MVP，架構上支援未來擴展成多物種（狗、兔、鳥、幻想生物等）。

### 目標使用者
- 已在 OpenClaw 生態內使用 agent 的玩家。
- 喜歡可愛、情緒陪伴、輕量養成體驗的使用者。
- 不想折騰安裝流程、希望「裝了就能玩」的使用者。

---

## 2) 產品目標與成功指標

### 產品目標
1. 讓使用者 5 分鐘內完成安裝並開始養第一隻寵物。
2. 讓 agent 能在對話中自然觸發寵物互動（餵食、玩耍、拍照、散步）。
3. 讓寵物有「可被辨識的個體差異」（至少 4 隻貓、不同性格與外觀）。
4. 保持與 catime 生態相容，方便共用角色資料與未來素材。

### MVP 成功指標（可觀測）
- 安裝成功率 > 90%（本機腳本 + OpenClaw skill）。
- 首次互動完成率（安裝後 10 分鐘內完成至少 1 次寵物命令）> 80%。
- 重複使用率（7 天內再次互動）> 35%。

---

## 3) 使用者體驗設計（MVP）

### 核心體驗流程
1. 使用者安裝 skill。
2. 第一次開啟時，系統引導選擇寵物（預設 4 隻貓）。
3. Agent 在聊天中可自動判斷何時觸發寵物能力。
4. 使用者可透過自然語句互動：
   - 「我的貓今天心情怎麼樣？」
   - 「幫我餵一下」
   - 「讓牠拍一張今天的照片」
5. 回傳寵物狀態 + 可發送圖片（先與 catime 相容，後續可接更多生成來源）。

### 寵物狀態模型（MVP）
- `mood`（心情）
- `energy`（精力）
- `hunger`（飢餓）
- `bond`（親密度）
- `last_interaction_at`（最後互動時間）

> 設計原則：狀態變化可感知、但不折磨使用者（偏輕養成，避免懲罰式機制）。

## 4) 創意方向（由 clawra + catime 延伸）

### A. 角色個性化（相容 catime）
- 直接借鑑 catime 的角色機制：
  - `characters/index.json`
  - `characters/<id>.json`
- 保留 catime 既有四貓（Momo / Mochi / Captain / Lingling）並映射為可養角色。

### B. 互動分層
- **日常互動**：餵食、摸摸、陪玩、休息。
- **敘事互動**：根據角色個性，回覆不同語氣。
- **視覺互動**：回傳寵物圖片（先支援 catime 圖源 + prompt 模板）。

### C. 未來可擴充為多物種
- 資料模型預留 `species` 與 `traits`。
- 角色檔格式統一，不綁定貓咪欄位。
- 未來新增物種只需新增 JSON 檔與素材，不改核心程式。

## 5) 技術方案總覽

### 5.1 架構分層
1. **Python CLI（clawpet）**
   - 管理寵物資料、狀態與互動回應。
   - 提供 skill 可呼叫的穩定命令介面。
2. **OpenClaw Skill（SKILL.md）**
   - 定義何時觸發、如何解析命令、如何回傳。
   - 利用 `metadata.openclaw.install` 提供低門檻依賴安裝。
3. **安裝腳本**
   - 一鍵複製 skill、更新 `~/.openclaw/openclaw.json`、注入 SOUL 能力段落。
4. **資料層（JSON）**
   - 角色檔與索引檔，採 catime 兼容結構。

### 5.2 與 catime 的相容策略
- 角色資料結構沿用 catime 的 `characters/*.json` 概念。
- 保留可解析 catime CLI 輸出的能力（URL、cat number、story）。
- 欄位命名盡量一致，降低跨專案共用成本。

### 5.3 與 clawra 的借鑑重點
- 借鑑其「安裝器 + SOUL 注入 + skill 快速上線」流程。
- 但產品內容轉為中性、普適、寵物陪伴導向。

## 6) 安裝與上手策略（重點）

### 安裝目標
- 使用者最好只要 1 個命令就能開始。

### MVP 安裝路徑
1. 本地開發安裝：
   - `./scripts/install_local.sh`
2. 未來發佈安裝（規劃）：
   - `clawhub install clawpet`
   - 或 `pip install clawpet` + 自動註冊 skill

### 低摩擦設計
- 自動建立 `~/.openclaw/skills/clawpet`
- 自動補齊 `openclaw.json` 的 skills 設定
- 自動注入 SOUL 寵物能力段落（重複安裝不重複注入）

---

## 7) MVP 功能邊界

### In Scope（本次要做）
- 可選擇與切換寵物（至少 4 隻貓）
- 可查詢寵物狀態
- 可執行基本互動（餵食/玩耍/休息）並改變狀態
- 可輸出圖片請求 prompt（供 OpenClaw 發送）
- OpenClaw skill + 本地安裝腳本

### Out of Scope（後續）
- 多人共享同一寵物宇宙
- 經濟系統/商城
- 跨裝置雲端同步
- 複雜 3D/即時動畫

---

## 8) 風險與對策

1. **OpenClaw 版本差異造成安裝失敗**
   - 對策：安裝腳本做版本檢查與清楚錯誤提示。
2. **CLI 輸出格式改動影響 skill 解析**
   - 對策：固定輸出欄位與測試案例，保持向後相容。
3. **寵物狀態設計過複雜導致勸退**
   - 對策：先做輕量核心狀態，逐步加深玩法。
4. **圖片來源不穩定**
   - 對策：先以 catime 相容流程為主，預留多來源 fallback。
